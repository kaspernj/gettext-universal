- Add `useLocale()` hook (`src/use-locale-expo.js`). Subscribes the calling component to locale changes and syncs the `TranslateContext`-set locale + Expo device locales into the global `config` so direct callers of `translate(...)` produce the same output the `useTranslate()` hook would produce — lets consumers declare `_ = translate` as a class field instead of assigning `this._ = useTranslate()` inside `setup()` (which types `this._` as possibly-undefined). Does not fight explicit `config.setLocale(...)` callers: writes are gated by refs that track the hook's own last-written values.
